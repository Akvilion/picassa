Структура проекту Pica3
========================

Потік даних в застосунку
=========================

1. Запуск:
   main.go → NewApp() → DefaultWindowConfig() → wails.Run()

2. Lifecycle:
   Startup() → DomReady() → [робота застосунку] → Shutdown()

3. Frontend → Backend:
   JavaScript → Wails Runtime → Go методи → повернення результату

4. Прозорість:
   Go (BackgroundColour: RGBA{A:0}) + CSS (background: transparent) = Прозоре вікно


Команди для роботи
==================

Розробка:
  make dev              Запустити у режимі розробки з hot reload
  wails dev             Альтернативний спосіб запуску

Збірка:
  make build            Зібрати production версію
  make build-debug      Зібрати debug версію
  wails build           Альтернативний спосіб збірки

Тестування:
  make test             Запустити тести
  make test-coverage    Запустити тести з покриттям
  go test ./...         Альтернативний спосіб тестування

Обслуговування:
  make clean            Очистити build директорію
  make fmt              Форматувати код
  make lint             Перевірити код лінтером
  make install-deps     Встановити залежності

Довідка:
  make help             Показати всі доступні команди


Ключові файли для редагування
==============================

1. Додати нову функцію:
   → internal/app/app.go

2. Змінити конфігурацію вікна:
   → pkg/config/window.go

3. Змінити UI:
   → frontend/dist/index.html
   → frontend/dist/style.css

4. Змінити прозорість:
   → frontend/dist/style.css (.glass-card background)

5. Додати нові команди:
   → Makefile


Принципи організації
====================

✓ Мінімальний main.go (тільки запуск)
✓ Бізнес-логіка в internal/
✓ Конфігурація в pkg/
✓ UI в frontend/
✓ Документація для всього
✓ Автоматизація через Makefile
✓ Git ignore для build та temp файлів
